# Задача 04 — Генерация MD-файла и отправка через Telegram-бот

## Цель

По готовому диаризованному JSON (`status = diarized`) сформировать Markdown-файл
с транскрипцией и диаризацией, затем отправить его пользователю через Telegram-бота
как документ.

---

## Входные данные

- `/var/data/transcripts/<job_id>_diarized.json`
- `telegram_chat_id` из таблицы `jobs`

---

## Требования к реализации

### Шаг A — Генерация MD-файла

Путь: `/var/data/results/<job_id>_transcript.md`

#### Шаблон файла

```markdown
# Транскрипция звонка

**Дата:** <YYYY-MM-DD>  
**Время:** <HH:MM> UTC  
**Длительность:** <MM:SS>  
**Спикеров:** <N>  
**ID задания:** #<job_id>  

---

## Диалог

**Спикер 1** · `00:00 – 00:09`

Добрый день, как дела с проектом?

---

**Спикер 2** · `00:09 – 00:18`

Всё хорошо, вчера закончили правки по баннерам.

---

**Спикер 1** · `00:19 – 00:35`

Отлично. Тогда можем двигаться к следующему этапу.

---
```

#### Правила форматирования

- Таймкоды выводить в формате `MM:SS` (если запись > 1 ч — `HH:MM:SS`)
- Каждая реплика отделяется горизонтальной чертой `---`
- Имя спикера — жирным, таймкод — в `backtick`
- Пустая строка между заголовком реплики и текстом

#### Имя файла для пользователя

`transcript_<YYYYMMDD>_<job_id>.md`

(используется как `filename` при отправке документа в Telegram)

### Шаг Б — Отправка через Telegram

Использовать `bot.send_document()`:

```python
from aiogram.types import BufferedInputFile

with open(md_path, "rb") as f:
    content = f.read()

await bot.send_document(
    chat_id=chat_id,
    document=BufferedInputFile(content, filename=filename),
    caption=(
        f"✅ Готово! Транскрипция звонка #{job_id}\n"
        f"📊 Длительность: {duration}\n"
        f"👥 Спикеров: {speakers_count}"
    ),
)
```

### Обновление статуса

- Начало: `status = sending`
- После успешной отправки: `status = done`
- При ошибке: `status = error`, записать `error_message`

## Критерий приёмки

1. Создать задание со статусом `diarized`, убедиться что `<job_id>_diarized.json` существует
2. Запустить воркер отправки
3. В Telegram пришёл `.md`-файл с корректным именем и подписью
4. Открыть файл: структура соответствует шаблону, таймкоды корректны, спикеры чередуются
5. Статус задания в БД стал `done`
6. Проверить на записи длиннее 60 минут: таймкоды в формате `HH:MM:SS`
