# Задача 01 — Telegram-бот: приём аудиофайла и очередь обработки

## Цель

Реализовать Telegram-бота на `aiogram` 3.x, который принимает аудиофайл от пользователя,
сохраняет его на диск и регистрирует задание в SQLite-очереди для последующей обработки.

---

## Входные данные

- Telegram-бот с токеном из переменной окружения `TELEGRAM_BOT_TOKEN`
- Аудиофайл, отправленный пользователем в личный чат с ботом

---

## Требования к реализации

### Обработчик входящих сообщений

Принимать сообщения типов:
- `voice` — голосовое сообщение
- `audio` — аудиофайл
- `document` — документ с аудиорасширением

Допустимые форматы: `mp3`, `mp4`, `m4a`, `wav`, `ogg`, `webm`, `flac`

Ограничения:
- Максимальный размер файла: **200 МБ**
- Если тип или размер не подходят — ответить пользователю сообщением об ошибке

### Сохранение файла

Путь: `<DATA_DIR>/audio/inbox/<timestamp>_<telegram_file_id>.<ext>`

- `<timestamp>` — UTC в формате `YYYYMMDD_HHMMSS`
- Папка создаётся при запуске если не существует
- При локальной разработке `DATA_DIR=./data`; на VPS `DATA_DIR=/var/data`

### Очередь заданий (SQLite)

База: `<DATA_DIR>/jobs.db`

Таблица `jobs`:

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INTEGER PK AUTOINCREMENT | Идентификатор задания |
| `file_path` | TEXT | Абсолютный путь к аудиофайлу |
| `original_filename` | TEXT | Оригинальное имя файла |
| `telegram_chat_id` | INTEGER | ID чата для отправки результата |
| `status` | TEXT | `pending` / `transcribing` / `transcribed` / `sending` / `done` / `error` |
| `created_at` | TEXT | ISO 8601 UTC |
| `updated_at` | TEXT | ISO 8601 UTC |
| `error_message` | TEXT | Текст ошибки если статус `error` |

### Ответ пользователю

После успешного сохранения:
```
✅ Файл получен и поставлен в очередь.
ID задания: #<job_id>
Формат: <ext> · Размер: <size> МБ
Ожидайте результат — это займёт несколько минут.
```

## Структура файлов

```
src/
├── bot/
│   ├── __init__.py
│   ├── main.py          ← запуск бота (polling)
│   ├── handlers/
│   │   ├── __init__.py
│   │   └── audio.py     ← обработчик аудио
│   └── db.py            ← инициализация SQLite, функции работы с jobs
└── config.py            ← переменные окружения
```

---

## Переменные окружения

| Переменная | Описание |
|------------|----------|
| `TELEGRAM_BOT_TOKEN` | Токен бота от @BotFather |
| `ALLOWED_CHAT_ID` | ID чата, с которого принимаются файлы (безопасность) |
| `DATA_DIR` | Корневая папка для данных (default: `./data` локально, `/var/data` на VPS) |

---

## Критерий приёмки

1. Запустить бота
2. Отправить голосовое сообщение → получить ответ с `job_id`, файл появился в `inbox/`, запись в БД со статусом `pending`
3. Отправить аудиофайл `.mp3` → аналогично
4. Отправить файл недопустимого формата (`.pdf`) → получить сообщение об ошибке, файл не сохранён
5. Отправить файл > 200 МБ → получить сообщение об ограничении
